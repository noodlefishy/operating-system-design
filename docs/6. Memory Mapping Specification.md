# RiSC-16 Memory & I/O Layout

With a 16-bit address space, the MVM has 65,536 available words. The OS partitions this space into logical segments.

## Memory Map (Suggested)

| Address Range       | Segment Name | Description                                      |
|:--------------------|:-------------|:-------------------------------------------------|
| **0x0000 - 0x000F** | **Vectors**  | Entry points for Boot and Interrupts.            |
| **0x0010 - 0x0FFF** | **Kernel**   | OS Code and Kernel Data structures.              |
| **0x1000 - 0xEFFF** | **Userland** | User process memory (Code + Heap).               |
| **0xF000 - 0xFEFF** | **Stack**    | System/User stack (Grows downwards from 0xFEFF). |
| **0xFF00 - 0xFFFF** | **MMIO**     | Hardware Device mapping.                         |

## Memory-Mapped I/O (MMIO)

Instead of dedicated "In/Out" instructions, devices are controlled by writing to specific addresses in the `0xFF00+`
range:

- **0xFF00**: **Console Output** (Writing a short here prints the char to terminal).
- **0xFF01**: **Console Input** (Reading here gets the last key pressed).
- **0xFF02**: **System Clock** (Reading here returns a millisecond timer).
- **0xFF03**: **Interrupt Acknowledge** (Used to clear the interrupt flag).
