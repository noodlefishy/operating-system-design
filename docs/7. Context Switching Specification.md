# Process Management & Context Switching

Because the RiSC-16 only has 8 registers, switching from Process A to Process B is extremely lightweight.

## The Process Control Block (PCB)

For every process, the Kernel maintains a block of memory (in the Kernel Segment) containing:

1. **Saved PC**: The address to resume from.
2. **Saved Registers**: R1 through R7 (7 words total).
3. **Saved SP**: The Stack Pointer location.

## The Switching Logic

When a Timer Interrupt occurs (Hardware Interrupt):

1. CPU jumps to `0x0001`.
2. Kernel code at `0x0001` performs the "Save State":
    - Stores R1-R6 into Process A's PCB in memory.
    - Moves R7 (which holds the interrupted PC) into memory.
3. Kernel logic decides which process is next (Scheduler).
4. Kernel performs the "Load State":
    - Loads R1-R6 from Process B's PCB.
    - Loads Process B's saved PC into R7.
5. Kernel executes `jalr r0, r7, 0` to jump back into Process B's code.

## Protection

- User processes should use the `MemoryManagementUnit` wrapper to ensure they cannot read/write into the **Kernel
  Segment (0x0000 - 0x0FFF)** or the **MMIO Segment**.
